核心技术实现

关键模块的技术方案与实现细节

1. 动态二维码凭证系统

• 生成策略：每次请求生成唯一 UUID

• 有效期：180秒（3分钟）

• 刷新机制：前端自动轮询，剩余60秒时提前刷新

• 安全性：票据与用户绑定，核销后状态更新

• 防复用：一经核销立即失效，不可重复使用

2. 实时位置追踪

• 轮询间隔：2秒

• 数据来源：司机端上报

• 存储方式：shuttle_status 表（支持 upsert）

• 缓存策略：1小时数据有效，超时自动清除

• 数据字段：经纬度、速度、航向、当前站点、距下站距离

3. JWT 认证体系

• 令牌类型：Bearer Token

• 有效期：24小时

• 角色识别：openid 前缀自动映射（admin_/driver_/默认乘客）

• 权限控制：API 级别中间件校验

• 设备绑定：支持 device_id 更新追踪

4. AI 智能调度建议

• 模型：Google Gemini 3 Flash Preview

• 输入：历史客流数据、时段满载率、排班信息

• 输出：JSON格式建议（标题、类型、描述）

• 建议类型：warning / success / info

• 降级策略：无 API Key 时返回预设模拟数据

业务流程详解

核心业务场景的完整流程

1 乘客乘车全流程

乘客登录

生成凭证

等待班车

扫码乘车

完成乘车

详细步骤：

  1. 乘客通过 OpenID 登录系统
  2. 前端调用 /api/tickets/generate 生成动态二维码
  3. 二维码有效期3分钟，前端自动轮询刷新
  4. 每2秒轮询 /api/shuttle/status 获取班车实时位置
  5. 司机扫描乘客二维码，调用 /api/tickets/verify 核销
  6. 系统记录核销日志，更新票据状态

驾驶中状态
